## Why this order
Each stage builds on the one before it — HTML/CSS fundamentals first, then layout systems, then a CSS framework, then JavaScript, then a JS framework. I'm not skipping ahead even if a topic looks more exciting, because gaps at the foundation show up later as bugs I can't debug.

## How I add new technologies later
When I move past React (TypeScript, Node.js, testing, Next.js, etc.), I add a new numbered top-level folder (`08-typescript`, `09-testing`, ...) following the exact same internal pattern as every folder before it: a `README.md` with notes + an `exercises/` folder. The structure doesn't need to change — it just grows.
