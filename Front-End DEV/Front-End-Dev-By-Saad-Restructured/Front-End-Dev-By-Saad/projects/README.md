# Projects

This is the folder recruiters will click first — every project here is **finished, working, and deployed** (or clearly marked "in progress").

## Categories
| Folder | What goes here |
|---|---|
| `01-mini-projects/` | Small single-concept builds (calculator, counter, quote generator) |
| `02-landing-pages/` | Static responsive landing pages (HTML/CSS/Bootstrap/Tailwind) |
| `03-javascript-projects/` | Interactive vanilla-JS apps (to-do list, weather app, quiz app) |
| `04-responsive-websites/` | Multi-page responsive sites |
| `05-react-projects/` | React apps and components |
| `06-clone-projects/` | UI clones built for practice (Netflix landing, Spotify UI, etc.) — clearly labeled as clones/practice, not original products |
| `07-portfolio/` | My actual personal portfolio site |

## Rule for every project folder
Each project must include:
1. A short **README.md** — what it does, why I built it, tools used
2. A **live demo link** (Vercel/Netlify/GitHub Pages) once deployed
3. **1–2 screenshots**
4. A **"What I learned"** section

> Projects are never mixed with learning notes. Notes live in the numbered tech folders (`01-html`, `02-css`, etc.).
